# API Integration Requirements — CPV Milestones & Awards

To support the updated **CPV Milestones & Awards** interface, the frontend requires updates to the backend API. Below are the specific requirements, data models, and endpoint designs needed from the backend team.

---

## 1. Global Configuration Endpoint: `GET /settings/cpv-milestones`

This endpoint should return all active milestone rules configured in the admin panel. 

### Current & Expected Rule Schema
Each rule needs to support **both** a cash award and an optional material award simultaneously.

```json
{
  "rules": [
    {
      "id": "e5357b52-3042-4ae5-898f-7fbf0d1f3786",
      "threshold": 10,
      "rewardType": "CASH",
      "rewardAmount": 400,
      "materialDescription": null,
      "name": "10 CPVs",
      "reward": "$400",
      "isActive": true
    },
    {
      "id": "...",
      "threshold": 5000,
      "rewardType": "BOTH", // Support CASH + MATERIAL
      "rewardAmount": 100, // $100 (₦100,000)
      "materialDescription": "Washing Machine worth $300 (₦300,000)",
      "name": "5,000 CPVs",
      "reward": "$100 + Washing Machine",
      "isActive": true
    }
  ]
}
```

## 2. User Milestone Progress Endpoint: `GET /earnings/cpv`

This endpoint must return the user's specific progress towards these milestones.

### Required Fields
We need the backend to return the user's personal/team breakdown, next milestone target, and achievement status for the rules:

* **`personalCpv`**: (float) CPV generated by the user's own purchases.
* **`teamCpv`**: (float) CPV generated by the user's downline/team.
* **`totalCpv`**: (float) Sum of personal and team CPV.
* **`requiredCpv`**: (float) The target CPV threshold of the next milestone.
* **`nextMilestoneName`**: (string) The name of the next milestone.
* **`cpvCashBonus`**: (float) Total cash bonuses earned from milestones.
* **`milestones[]`**: List of all milestones showing the user's progress.
